# Гайд по оформлению отчетов

В этой папке находится эталонный пример оформления лабораторной работы. Мы используем **Typst** вместо Word/LaTeX, потому что это быстрее, удобнее и позволяет верстать отчеты кодом, сохраняя идеальную структуру.

## 📁 Структура папки отчета

Чтобы не превращать проект в свалку, придерживайтесь следующей структуры (см. папку `example`):

- `лрX.typ` — основной файл отчета. Все изменения текста и логики вносятся сюда.
- `figures/` — все графические материалы.
  - `bs/` (block-schemes) — папка для блок-схем.
  - `ss/` (screenshots) — скриншоты работы программы или консоли.
- `scripts/` — переиспользуемые модули Typst (например, для сложных таблиц).
- `title.pdf` — нормальный титульник *(надеюсь, что временно)*, потому что в Typst он неверный

## 🚀 Быстрый старт

1. **Установите инструменты:**
    - [Typst](https://github.com/typst/typst/releases/latest) (либо используйте пакетный менеджер).
    - **Обязательно:** Установите расширение **TinyMist** в VS Code.
2. **Скопируйте пример:** Возьмите содержимое папки `example` за основу для своей новой лабы.

## Полезные фишки в этом шаблоне

### 1. Авто-ГОСТ (`modern-g7-32`, [typst-gost](https://typst-gost.ru/))

Мы используем пакет `@preview/modern-g7-32`. Он автоматически делает правильные отступы, шрифты (Times New Roman), нумерацию разделов и формул. Вам не нужно возиться с линейкой в Word.

### 2. Многостраничные таблицы

В `scripts/table_multipage.typ` лежит функция `table-multi-page`.
Эта функция сделана для того, чтобы разбиение таблицы на несколько страниц соответствовало ГОСТу, хотя для Ерофеева это не особо нужно

### 3. Подключение кода

Вместо того чтобы копировать код в отчет вручную, используйте функцию `read`:

```typst
#let code = read("../main.c")
#raw(code, block: true, lang: "none")
```

Так ваш отчет всегда будет содержать актуальную версию кода из соседней папки.

### 4. Приложения и Блок-схемы

В примере показано, как развернуть страницу в альбомную ориентацию (`flipped: true`) специально для больших схем, чтобы они оставались читаемыми.

## 💡 Советы по оформлению

- **Имена файлов:** Не используйте пробелы в названиях картинок (лучше `sort_logic.png`, чем `картинка 1.png`).
- **Скриншоты:** Делайте скриншоты в **светлой** теме.
- **Пути:** Используйте относительные пути для вставки изображений: `image("figures/ss/result.png")`.

### Как скомпилировать?

Если не используете TinyMist, выполните в терминале:

```bash
typst compile лр1.typ
```
