# Общий конспект лекций по Алгоритмам и Структурам Данных

## Лекция 1. Определение алгоритма и алгоритмического языка

### Основные понятия
#Алгоритм #Алгоритмическая_модель
**Алгоритм** — конечный набор правил, однозначно раскрывающих содержание и последовательность операций, выполняемых для решения поставленной задачи.

#Алгоритмизация
Для построения алгоритма выполняется ==алгоритмизация==, которая заключается в:
- разделении процесса решения задачи на отдельные взаимосвязанные этапы;
- формальной записи содержания этих этапов;
- определении порядка их выполнения.

#Формализация
Под **формальной записью** (==формализацией==) подразумевается описание на формальном языке (*язык математики или логики*), объективность синтаксиса которого делает описание безусловно понятным и однозначно трактуемым.

#Алгоритмический_процесс
Реализация алгоритмической модели на уровне программирования (кодирования) называется **алгоритмическим процессом**.

### Структуры алгоритмов
Алгоритм любой сложности может быть представлен **комбинацией 3 структур**:
1. Следование (линейная последовательность блоков).
2. Разветвление (если-то-иначе).
3. Цикл (предусловие `while`, постусловие `do-while`, счетный `for`).

### Свойства алгоритма
#Операнды #Операции
Объекты, над которыми мы работаем, называются **операндами**.
Действия, которые мы производим над операндами — **операции**.

**Свойства алгоритма**:
- **Дискретность**. Вся процедура расчета должна быть представлена в виде отдельных действий (операций).
- **Детерминированность** (однозначность). Порядок и действия должны быть однозначно понятны.
- **Результативность** (конечность). Алгоритм должен заканчиваться за конечное число шагов.
- **Адекватность** (правильность). Алгоритм должен выдавать ожидаемый (правильный) результат.
- **Массовость**. Алгоритм должен быть применим к предполагаемому множеству входных данных.
  > **Множество входных данных** — это полный спектр (домен) допустимых значений и параметров, с которыми алгоритм должен уметь корректно работать. Алгоритм создается не для решения одной узкой задачи (например, сложить $2+2$), а для решения целого класса подобных задач (сложить $A+B$, где $A$ и $B$ входят в множество допустимых чисел).

## Лекция 2. Трансляция кода и работа компилятора

### Трансляция, интерпретация и компиляция
- **Интерпретатор** — транслирует каждый оператор в промежуточный код и сразу выполняет его.
- **Компилятор** — преобразует весь текст программы (исходный модуль) в машинный язык (объектный модуль).

**Этапы компиляции:**
1. **Лексический анализ (сканирование)**. Превращение сплошного текста программы в цепочку значимых единиц — лексем (токенов).
   > *Пример под капотом (из слайдов):* 
   > Текст `x = a + 5;` превращается в список кодов:
   > 1. `ID` (ссылка на 'x')
   > 2. `OP_ASSIGN` (знак равно)
   > 3. `ID` (ссылка на 'a')
   > 4. `OP_PLUS` (плюс)
   > 5. `CONST` (число 5)
2. **Синтаксический анализ (парсинг)**. Формирование синтаксического дерева разбора.
3. **Семантический анализ**. Интерпретация по смыслу (проверка типов). Из дерева разбора строится *дерево операций*.
4. **Распределение памяти**. Привязка к виртуальным/физическим адресам.
5. **Генерация и оптимизация кода**.

## Лекция 3. Классификация структур данных и Память

### Глобальная классификация структур данных (из слайдов)
1. **Линейные:** массив, список, связанный список, стек, очередь, хэш-таблица.
2. **Иерархические:** двоичные деревья, n-арные деревья, иерархический список.
3. **Сетевые:** неориентированный граф, ориентированный граф.
4. **Табличные:** таблицы реляционной базы данных, двумерный массив.

В линейных структурах элементы располагаются последовательно. Виды доступа: *индексный*, *прямой* (по смещению), *последовательный* (переход по указателям).

### Реализация стека (Stack)
Стек работает по принципу **LIFO** (Last In First Out). 

**Реализация стека на базе массива:**
```cpp
const int MAXSIZE = 100;
struct Stack {
    char data[MAXSIZE];
    int size;
};

int Push(Stack &S, char value) {
    if(S.size == MAXSIZE) return 0; // Переполнение
    S.data[S.size] = value;
    S.size++;
    return 1;
}

char Pop(Stack &S) {
    if(S.size == 0) return char(255); // Пусто
    S.size--;
    return S.data[S.size];
}
```

**Реализация стека на базе связного списка:**
```cpp
struct Node {
    char data;
    Node *next;
};

void Push(Node* &Head, char value) {
    Node *NewNode = new Node;
    NewNode->data = value;
    NewNode->next = Head;
    Head = NewNode;
}

char Pop(Node* &Head) {
    if(Head == NULL) return char(255);
    Node *q = Head;
    char value = Head->data;
    Head = Head->next;
    delete q;
    return value;
}
```

## Лекция 4. Очереди (Queue) и Деки (Deque)

Очередь работает по принципу **FIFO** (First In First Out).

### Базовая реализация очереди на массиве (ползет по памяти)
```cpp
struct Queue {
    int data[N]; // Массив данных
    int last;    // Указатель на конец очереди
};
void Add(Queue *Q) {
    if (Q->last == N) return; // Очередь заполнена
    // ... ввод value ...
    Q->data[Q->last++] = value;
}
void Delete(Queue *Q) {
    // При удалении все элементы сдвигаются влево (O(n) сложность)
    for (int i=0; i < Q->last; i++) 
        Q->data[i] = Q->data[i+1];
    Q->last--;
}
```

### Кольцевая очередь (Оптимальная)
Решает проблему смещения массива ($O(1)$ на удаление). Указатели `first` (голова) и `last` (хвост) бегают по кругу.
**Математика сдвига по кольцу:** `(Q->last % N) + 1` (остаток от деления).

```cpp
struct Queue {
    int data[N];
    int first; // Указатель на начало
    int last;  // Указатель на конец
};
// Проверка на пустоту
bool Empty(Queue *Q) {
    return (Q->last == Q->first);
}
// Добавление (хвост догоняет голову)
void Add(Queue *Q, int value) {
    if (((Q->last % (N-1)) + 1) == Q->first) {
        cout << "Очередь заполнена"; 
    } else {
        Q->data[Q->last] = value;
        Q->last = (Q->last % (N-1)) + 1; // Круговой сдвиг
    }
}
```

#Дек #Deque
**Дек (DECK)** — двусторонняя очередь. Вставка и удаление возможны с обоих концов (`AddL`, `AddR`, `DelL`, `DelR`).

## Лекция 5. Связные списки и Итераторы (STL)

### Работа с узлами в Связном списке (Linked List)
При добавлении или удалении элементов в связном списке самое важное — **не потерять указатели**.

**Удаление заданного узла (`p`):**
Необходимо найти адрес *предыдущего* узла (`q`), чтобы перекинуть его указатель:
`q->next = p->next; delete p;`

**Добавление узла перед заданным:**
Либо находим предыдущий узел, либо используем хитрость: вставляем новый узел *после* текущего, а потом просто меняем их значения (data) местами.

### Итераторы и их отличия
В C++ для перебора коллекций используются разные подходы. Важно понимать разницу между тремя видами "итераторов" (способов обхода):

1. **Классический цикл `for` (с индексом или арифметикой указателей):**
   - Используется для массивов или векторов (`vector`).
   - Поддерживает **все арифметические операции** (случайный доступ): `it + 5`, `it += 2`, обращение по индексу `arr[i]`.
2. **Итератор коллекций (например, `list::iterator`):**
   - Для двусвязных списков (`list`) элементы разбросаны в памяти, поэтому случайный доступ невозможен.
   - Поддерживает **только инкремент и декремент** (`++`, `--`). Нельзя сделать `it + 5`, для смещения используется функция `advance(it, 5)`.
3. **Цикл Range-based `for` (или `for_each`):**
   - Синтаксис: `for (auto element : collection)`
   - **Обязательный проход по всей коллекции** от начала до конца (по умолчанию). Вы абстрагируетесь от индексов и указателей, просто получая каждый элемент по очереди.

### Таблица базовых методов STL (vector и list)
| Метод | Описание |
|---|---|
| `size()`, `empty()` | Размер контейнера / проверка на пустоту |
| `push_back(val)`, `pop_back()` | Добавить/Удалить элемент в конце |
| `push_front(val)`, `pop_front()` | Добавить/Удалить в начале (только для `list` и `deque`) |
| `begin()`, `end()` | Итератор на начало / на конец (за последним элементом) |
| `insert(it, val)` | Добавить элемент в позицию, на которую указывает итератор `it` |
| `erase(it)` | Удалить элемент в позиции итератора `it` |

## Лекция 6. Поиск подстрок

Поиск шаблона $P$ (длиной $m$) в тексте $T$ (длиной $n$).

### Наивный поиск
Посимвольное сравнение. Если несовпадение — шаблон сдвигается ровно на 1 символ вправо и всё начинается заново.
Общая максимальная сложность: $O((n-m+1) \cdot m) = O(m \cdot n)$.

### Метод Бойера-Мура
Алгоритм сравнивает символы **справа налево** (с конца шаблона). 
При несовпадении применяются правила для максимального сдвига (чтобы не двигать на 1 шаг, как в наивном).

**Правило плохого символа (Bad Character Rule):**
Строится "Таблица плохих символов" `BadTable`. Для каждого символа алфавита в ней записана его последняя позиция в шаблоне.
Если обнаружено несовпадение (в тексте встретился символ `W`, которого нет в шаблоне или он стоит левее), шаблон сдвигается по формуле:
`Сдвиг = i + j - BadTable(w)`
Если символа вообще нет в шаблоне (`BadTable(w) == 0`), шаблон сдвигается целиком за этот плохой символ.

### Алгоритм Кнута-Морриса-Пратта (КМП)
Сложность $O(n)$.
Основан на **Префикс-функции**. Мы ищем длину максимального бордера (подстрока, которая одновременно является и префиксом, и суффиксом). Это позволяет алгоритму знать, какую часть шаблона уже не нужно проверять повторно при сдвиге.

## Лекция 7. Хэширование и Хэш-таблицы

#Хэширование #Хэш_таблица
**Хэширование** — преобразование входных данных в битовую строку фиксированной длины (хэш).
**Хэш-таблица** — массив (или массив списков), где позиция элемента вычисляется по формуле: `index = hFunc(key)`.

> *Визуально (из слайдов):* Ключи ("Жираф", "Слон") проходят через "Hash Function", которая выдает числа (05, 127). Это число становится индексом массива, куда кладется значение.

Коэффициент заполнения: $\alpha = \frac{n}{h}$ (занято / размер таблицы). В идеале операции работают за $O(1)$.

### Свойства хорошей хэш-функции
- Быстрота вычисления (не зависит от числа элементов, $O(1)$).
- Детерминированность (для одного ключа всегда один хэш).
- Равномерное распределение по индексам.
- Минимизация коллизий.

### Разрешение коллизий
**Коллизия** — когда двум разным ключам выпадает один и тот же индекс.
1. **Открытая адресация**: Если ячейка занята, ищем следующую свободную (линейно `i+1`, `i+2` или квадратично). *Минусы:* появляются длинные "слипшиеся" куски занятых ячеек, при удалении нужно ставить специальные метки-заглушки.
2. **Метод цепочек (закрытая адресация)**: В ячейках массива лежат не сами элементы, а указатели на Связные списки. Все элементы с одинаковым хэшем просто добавляются в список по этому индексу.
